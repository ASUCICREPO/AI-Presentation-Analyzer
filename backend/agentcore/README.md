# AgentCore ARM64 Container

This directory contains the AgentCore runtime container that gets automatically built and deployed to ECR when you run `cdk deploy`.

## How It Works

The CDK stack uses `DockerImageAsset` to automatically:
1. Build the Docker image for ARM64 architecture
2. Push it to a CDK-managed ECR repository
3. Reference it in the AgentCore Runtime

No manual build steps required!

## Deployment

```bash
cd backend
npm run cdk deploy
```

CDK will automatically:
- Build the ARM64 container image (using Docker Buildx if needed)
- Push to ECR
- Deploy the AgentCore Runtime with the new image

## Architecture

- **Platform**: `linux/arm64` (required by AgentCore)
- **Base Image**: `python:3.12-slim`
- **Build**: Handled automatically by CDK
- **Registry**: CDK-managed ECR repository

## Making Changes

1. Edit files in `backend/agentcore/`
2. Run `npm run cdk deploy` from `backend/`
3. CDK rebuilds and redeploys automatically

That's it!

## Files

- `Dockerfile` - Container definition
- `index.py` - AgentCore runtime entry point
- `requirements.txt` - Python dependencies
- `.Dockerignore` - Files to exclude from build

## Notes

- The image is built locally using Docker (or Docker Buildx for cross-platform builds)
- CDK handles all ECR repository management
- Image tags are automatically generated by CDK
- No manual ECR push commands needed
